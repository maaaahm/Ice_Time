# Ice_Time

## Description
Ice Time is a personal development project designed to help members of the Cincinnati Adult Hockey League (CAHL) quickly identify schedule changes and avoid missed games.

The long-term goal is to provide near real-time notifications when game schedules are modified, reducing the need for players to manually monitor league websites or email updates.

This project serves as both a practical tool for hockey players and a platform for exploring secure application development practices.

## Objectives
The initial version of Ice Time focuses on identifying schedule changes for upcoming hockey games.

Planned functionality includes:
  - Notification of game time changes
  - Notification of game location changes
  - Detection of scheduling conflicts for players participating on multiple teams
  - Push notifications delivered to mobile devices
  - User-configurable notification preferences

## Why This Project?
As a player in the CAHL, I've noticed that messaging regarding schedule changes come at irregular times, sometimes resulting in players missing games altogether. Messaging is also easy to miss as it gets mixed into team chats within BenchApp, our team management application. This project aims to improve visibility into those changes while providing a real-world application to practice software engineering and cybersecurity concepts.

Development is being performed using AI-assisted tools, including Cursor, as a way to accelerate learning and expose myself to new development patterns. A key objective is to critically evaluate generated code, understand its functionality, identify potential security concerns, and implement improvements where appropriate.

## Security Goals
Security considerations are included throughout development rather than being added later.

Areas of focus include:
  - Secure handling of user information
  - Authentication and authorization fundamentals
  - Input validation and data sanitization
  - API security practices
  - Dependency management and vulnerability monitoring
  - Logging and monitoring considerations
  - Secure software development lifecycle (SSDLC) principles
  - Security review of AI-assisted code generation

As the project evolves, security decisions and lessons learned will be documented within this repository.
